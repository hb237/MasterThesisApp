Start and Stop

In the page General settings there are a green play and a red stop discard button. Before launching a new monitoring instance the user must save the changes they made to the settings. If they later on decide to stop the application it will delete all intermediate data structures.

General settings

Below you can find a description of each configuration parameter:

  • Dashboard refresh rate:
    The rate that defines how often a AJAX query will fetch data from the server. The value is the time in seconds in between two requests.
  • Start block:
    The lowest block number that the application should still include into the monitoring.
  • End block:
    The highest block number that the application should still include into the monitoring.
  • Monitoring window block size:
    The maximum number of blocks that will serve as an input for the monitoring.
  • Confirmation blocks
    Number of most recent blocks the monitoring application should not process.
  • Data input mode:
    Select whether the application should work with test data or live streaming data.
  • Example data set:
    This setting applies if the data input mode defines to work with test data. The user can select the test data set they want to use.
  • Replay speed:
    This setting applies if the data input mode defines to work with test data. The user can accelerate or slow down the replay of the blocks in the tas data set. Numbers above 1 accelerate and below 1 slow down the replay.
  • Noise threshold:
    When creating BPMN diagram the noise parameter defines how many activities the algorithm includes into the model. The values range between 0 and 1.
  • Geth node IP address:
    The IP of the Geth node the application should connect to.
  • Geth node port:
    The Port of the Geth node the application should connect to.
  • Convert ETH to currency:
    The currency the dashboard prices should be displayed in. It converts costs in Ether in to the current market price for the selected currency.
  • Monitoring status
    The monitoring status shows whether the monitoring process is currently running or if it is stopped.

BLF manifest

The manifest editor provides the user with a simple option to edit, validate and save BLF manifest to the server. One can also download from the server the currently stored manifest or upload an already existing manifest from the local machine.

BPMN Model

The BPMN editor provides the user with an appealing editor to create simple BPMN diagrams and then save them to the server. It provides only a small subset of all BPMN elements, du to pm4py's limitations. One can also download from the server the currently stored BPMN model or upload an already existing BPMN model from the local machine.
